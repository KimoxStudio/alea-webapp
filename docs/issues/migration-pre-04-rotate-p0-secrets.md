# Pre-04: Investigate AUTH_SESSION_SECRET + document P0 secret rotation checklist
Phase: Pre (Supabase to Neon migration blockers). Depends on: none.
Problem: before migration/deploy, all .env.local secrets need rotation: service-role key, CRON_SECRET, AUTH_SESSION_SECRET, QA creds. Separately, AUTH_SESSION_SECRET appears to have no code consumer and may be dead config. Actual secret rotation is a user-only action -- no agent may perform it. This issue covers only the code-side investigation and produces a checklist for the user to execute manually.
Scope: (1) Grep the codebase for every reference to AUTH_SESSION_SECRET, determine whether anything reads/uses it -- if unused, propose removing it from .env.example/schema files and note as dead config; if used, document where/why. (2) Produce a rotation checklist (PR description or a doc under docs/) listing which secrets need rotating, where currently set (env var NAME only, never value), and what depends on each (so user knows what breaks if rotated without updating dependents). (3) Do not modify .env.local, .env.example values, or any actual secret. Do not print secret values anywhere.
Acceptance criteria: clear finding on whether AUTH_SESSION_SECRET is used, with file:line evidence. If unused, a small PR removing it from example/schema files (not .env.local). A rotation checklist artifact the user can follow step by step.
Out of scope: do not rotate any actual secret value. Do not touch Supabase/Vercel dashboard configuration.
